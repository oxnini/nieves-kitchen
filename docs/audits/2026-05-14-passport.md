## Fixes applied during audit

### [P1] Cancellation postmarks were vertically misplaced on non-square visas

`components/passport/CountryStampSlot.tsx:209-210`. Previous math used `left: (cx-23)% / top: (cy-23)%` to center a square cancellation. `top%` resolves against the wrapper's height while the cancellation is sized off the wrapper's longest edge, so on landscape visas the postmark drifted below center by `0.23·(W−H)` units. Replaced with `left:cx%, top:cy%, transform: translate(-50%, -50%)` so centering is correct independent of aspect.

### [P1] Focus restoration on passport close

`components/passport/PassportModal.tsx:23-31, 49-53`. React fires child effects before parent effects, so when `BookletShell.useEffect` focused the close button at mount, it ran before `PassportModal.useEffect` could call `document.activeElement` — meaning the modal saved the close button as "previously focused" and on close tried to focus an unmounting node. Captured `document.activeElement` during render via `useState` lazy initializer (runs in parent render, before any child render or effect), and switched the cleanup to read that captured value.

## Item 1 (followup) — SPEC §9 drift + RegionHalf left-shift

**Done.** Long thread, ended at option B + center alignment.

- `lib/passport-pack.ts` has `HALF_CAPACITY = 4` because image visas at 1.4× wrap to 2-per-row in flex (per leading comment). SPEC §9.1 incorrectly claimed 3×4=12-per-half grid; §9.2 proposed a 2-col migration whose math doesn't survive contact with the 1.4× image visa multiplier (image visas would wrap to 1-per-row, violating §9.6's "not a gallery view" rule).
- Patched `docs/stamps/SPEC.md`: §9.1 now describes shipped hybrid flex-wrap reality; §9.2 retitled "Design intent" and explains why `IMAGE_STAMP_SIDE = 1.4` is the ceiling; §9.5 rewritten as a knob-by-knob reference. §9.3/§9.4/§9.6 untouched.
- `components/passport/RegionHalf.tsx:35`: tried padding adjustment (overcompensated — landscape visa pairs no longer fit, forcing vertical overflow), reverted to `justifyContent: 'center'`. Trade: lone trailing stamps center in their row instead of anchoring left. Documented in SPEC §9.1.

### P2

**[7] Nested cards (anti-pattern)** — done.
Replaced both with the hairline grammar already used elsewhere in these files (`RecapRow`'s dotted bottom rule, ContentsSpread's region-list dotted rows).
- `InsideFrontSpread` Stat trio: dropped `bg-brown-dark/5 rounded-xl` tiles for a top/bottom hairline frame (`border-t border-b border-brown-light/40`) with `border-l` column rules between stats; reads as an editorial stat strip, not three cards.
- `ContentsSpread` recommendation tiles: dropped `rounded-xl bg-brown-dark/5 ... border` for `py-2 border-b border-dotted border-brown-light/50 hover:border-terracotta/40`, matching the contents list above. Also flattened the nested `flex-1 justify-center` wrapper that only existed to center text inside the old `items-stretch` row; now `items-center` does it without extra divs. Title gets `group-hover:text-terracotta` so the affordance is on the text, not the tile.

**[8] Three independent focus-trap implementations** — done.
Extracted `useFocusTrap(ref, { active?, onEscape?, autoFocus? })` to `components/passport/hooks/useFocusTrap.ts`. Onescape is held in a ref so callers can pass a fresh closure each render without re-binding the listener. Refactored all three call sites:
- `PassportModal` — dropped the React `onKeyDown` Tab handler and the document-level Escape effect.
- `PassportHelpModal` — collapsed the open-gated effect; trap activates on `active: open`.
- `StampedRecipesModal` — dropped the local effect.

The selector now uses `:not([disabled])` on inputs to match Help/StampedRecipes prior behavior while still covering PassportModal's broader set.

**[9] `useBookletNav` initial-load effect only fires on `spreads.length` change** — done.
Replaced the bare `eslint-disable` with a comment explaining the intent: depend on length, not identity, so the effect stays an initial-load sync and doesn't re-run on every pack identity change. Documented the same-length/different-slug-order trade-off the audit called out.

**[10] Close button in `StampedRecipesModal` doesn't use `InkMark` vocabulary** — done.
Swapped the plain `<X>` button for an `InkMark` with the same `X strokeWidth={1.5}` glyph the booklet's `CloseInkMark` uses. Didn't reuse `CloseInkMark` directly because that variant is bound to the passport modal's own `onClose` semantics; the stamped modal has its own dismiss handler.

**[11] `RegionChipStrip.scrollIntoView` can scroll the page on desktop** — done.
Bailed early when `strip.scrollWidth <= strip.clientWidth` so the call only fires on the mobile rail where the strip actually overflows.

### P3

**[12] Duplicated `COLS_PER_HALF` constant** — evaluated, not merged.
SPEC §9.5 (post-reconciliation) treats `COLS_PER_HALF = 3` (sizing reference, `BookletShell`) and `HALF_CAPACITY = 4` (packing cap, `lib/passport-pack`) as deliberately separate knobs in separate modules — they're different numbers in different domains. Physically merging them into one file would couple two unrelated concerns. Added a cross-reference comment at the `COLS_PER_HALF` declaration that names `HALF_CAPACITY`, explains why capacity ≠ columns, and points to SPEC §9.5 so a future reader finds both knobs together.

**[13] Loading state typography** — done.
Both loading states now share a `BookletLoading` component: short hairline rule above and below a `font-stamp text-[11px] tracking-[0.32em] uppercase` label, rendered inside `passport-light` so it picks up parchment tokens. Replaces the generic `text-brown-medium` body text.

**[14] `PassportOverlay` Suspense fallback duplicates the booklet's own loading state** — done.
`BookletLoading` takes a `variant: 'shell' | 'data'` prop. The Suspense fallback (route-hook suspense) reads "Unfolding the booklet"; the TanStack data state reads "Opening your passport". Same visual, different label — distinguishable in the wild without changing the rest of the chrome.

**[15] `TierLedger` `useEffect` deps include the module-level `SEGMENTS` constant** — done.
Dropped `SEGMENTS` from `TierLedger.tsx:145`'s deps array. The module-level constant has a stable reference; including it added nothing and read as a confused eslint pacification.

## Systemic issues

- **Three focus-trap implementations** — extract.
- **Three modal-backdrop colors** — pick one per nesting level or commit to a single value.
- **SPEC vs code drift** — §9 reconciled. Watch this pattern recurring as the SPEC continues to evolve ahead of code.
- **Card nesting in editorial spreads** — every screen wants to be a card, but a passport spread is already a card. The instinct to wrap stats and recommendations in their own `rounded-xl` tiles is the AI dashboard pattern `impeccable` warns against.

## Positive findings

- Visa + dated cancellation grammar is genuinely original, well-documented in `docs/stamps/SPEC.md`.
- Aspect-ratio-aware stamp sizing (`IMAGE_STAMP_SIDE * sqrt(aspect)`) is a careful detail most projects skip.
- Anonymous Supabase session gating via `useSessionReady` keeps the query graph clean.
- `PassportAffordance` prefetch (idle + hover + focus) is a thoughtful implementation of CLAUDE.md's "preload assets behind a click" rule.
- The journey path in `TierLedger.JourneyMap` (sampled SVG path → progress fill) is overengineered in the best way.
- Sepia overrides via `.passport-paper` / `.passport-light` are disciplined.
